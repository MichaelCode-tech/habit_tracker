# Contributing to Habit Tracker

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to Habit Tracker. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues to see if the bug has already been reported.
* If not, open a new issue. Include a clear title and as much relevant information as possible, such as:
    * The version of the app you're using.
    * Steps to reproduce the bug.
    * Expected behavior.
    * Screenshots if applicable.

### Suggesting Enhancements
* Open a new issue with the label "enhancement".
* Explain the feature you'd like to see and why it would be useful.

### Pull Requests
* Fill in the pull request template (if provided).
* Follow the existing code style.
* Ensure your changes do not break existing functionality.
* Include screenshots or GIFs if your PR affects the UI.

## Development Setup

1.  Follow the instructions in the `README.md` to get the project running locally.
2.  Make sure you run `dart run build_runner build` if you modify any Isar models.
3.  Format your code using `flutter format .` before committing.

## Style Guide

* Use meaningful variable and function names.
* Keep functions small and focused on a single task.
* Comment your code where the logic is complex.

Happy coding! 🌿
